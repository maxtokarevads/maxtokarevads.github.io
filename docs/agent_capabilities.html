Google Ads

Search · PMax · Shopping · Display · YouTube · App

6 режимів
📋
Plan — Планування кампанії
Повний план запуску від структури до KPI
/plan · platform: google
  • Вибір типу кампанії: Search / PMax / Shopping / Display / YouTube / App
  • Структура Campaign → Ad Group → Ad з логікою розбивки
  • Стратегія ставок: tCPA / tROAS / Maximize Clicks з порогами конверсій
  • Таргетинг: ключові слова + аудиторії + демографія + пристрої
  • B2B Офлайн-конверсії, GCLID, оптимізація на MQL/SQL з CRM
  • Ecom PMax з Asset Groups, Shopping feed, бренд-захист
  • Learning phase: мінімум конверсій, заморозка змін, моніторинг
🔬
Analyze — Діагностика кампанії
Пошук першопричин і план дій
/analyze · platform: google
  • Розбір метрик: CTR, CPC, CPA, ROAS, QS, Impression Share
  • Виявлення проблем: низький QS, Learning Permanent, PMax-канібалізація бренду
  • Порівняння з бенчмарками (Search CTR ≥3%, QS ≥7, IS ≥70%)
  • Аналіз Lost IS — по бюджету vs по рейтингу окремо
  • Прогноз покращення після рекомендацій
  • Рівень впевненості: High / Medium / Low для кожної рекомендації
  • Режим вільного тексту — аналіз без структурованих метрик
✍️
Copy — Рекламні тексти
RSA з перевіркою символів і assets
/copy · platform: google
  • 15 заголовків ≤30 символів: keyword / USP / CTA / urgency / brand
  • 4 описи ≤90 символів: pain→solution / benefit+CTA / proof / offer
  • Pinning: що закріпити на позиції 1 і 3
  • 4 Sitelinks (≤25 chars + 2×35 chars desc) + 4 Callouts + Snippets
  • Dynamic Keyword Insertion — приклади {`{Keyword:Default}`}
  • Авто-підрахунок символів [XX chars] + фінальний checklist
🎯
Retargeting — Ремаркетинг
RLSA · Display · Customer Match · DRA
/retargeting · platform: google
  • RLSA по сегментах: всі відвідувачі +20% → кинуті кошики +80%
  • Display ремаркетинг: sequential, frequency cap 5–7/день
  • Customer Match з CRM (мін. 1 000 збігів) для upsell / exclusion
  • Ecom Dynamic Remarketing — точний товар що переглянули
  • Стратегія по лійці: MOFU (нагадування) → BOFU (urgency + знижка)
  • Аудиторний membership duration: 7 / 14 / 30 / 90 днів
⚗️
A/B Test — Тестування
Ad Variations · Drafts & Experiments
/ab_test · platform: google
  • Ad Variations для RSA headline тестів (50/50 авто-спліт)
  • Drafts & Experiments: стратегія ставок, таргетинг, PMax vs Search
  • Розрахунок sample size: n = 16×p×(1−p)/δ² (95% confidence)
  • Таблиця мінімального трафіку для CVR 1–10% з MDE 10/20/30%
  • 5 пріоритизованих тестів: бід → headline → match type → LP → PMax
  • Правила: 1 змінна, не зупиняти рано, документ hypothesis→result

Keyword universe · Аудиторії · Конкуренти

  • SKAG vs STAG: коли використовувати кожен підхід
  • In-market, custom intent/affinity аудиторний лейер

Meta Ads

Facebook · Instagram · Reels · Stories · Lead Forms

6 режимів
📋
Plan — Планування кампанії
Objective · структура · формати · ставки
/plan · platform: meta
  • Вибір Objective: Awareness / Consideration / Conversion / Catalog Sales
  • Ecom Advantage+ Shopping Campaign (ASC)
  • Структура Campaign → Ad Set → Ad, CBO vs ABO з логікою вибору
  • Таргетинг: Core / Custom Audiences / Lookalike 1–7% / Advantage+
  • Формати: Image, Video, Carousel (10 cards), Collection, Stories/Reels, Lead Form
  • Стратегія ставок: Lowest Cost → Cost Cap → ROAS Target
  • Creative cadence: 10–20 нових assets/тиждень, ротація при frequency >3–4
🔬
Analyze — Діагностика кампанії
Воронка · fatigue · attribution · Learning
/analyze · platform: meta
  • Метрики: CPM, CTR, CPC, CPA, ROAS, CVR, Frequency, Hook Rate, Video CR
  • Creative fatigue: Frequency >3 + CTR drop + CPM rise pattern
  • Learning Limited: причини і як вийти (бюджет / конверсії / структура)
  • Attribution inflation: Meta ROAS vs GA4 gap >30% — cross-validation
  • Воронковий діагноз: CPM → CTR → CVR → ROAS (де розривається)
  • Бенчмарки 2026: CTR ≥0.9%, Hook Rate ≥25%, Frequency ≤5/7d
  • Режим вільного тексту без структурованих метрик
✍️
Copy — Рекламні тексти
3 hook-варіанти · TOFU/MOFU/BOFU
/copy · platform: meta
  • Варіант 1 — Hook: Pain / Problem (відкриваємо болем)
  • Варіант 2 — Hook: Number / Result (конкретна цифра або результат)
  • Варіант 3 — Hook: Question / Intrigue (питання на яке хочуть "так")
  • Ліміти: Primary Text ≤125 / Headline ≤40 / Description ≤30 символів
  • Funnel stage (TOFU/MOFU/BOFU) змінює tone і підхід тексту
  • Video Script першої секунди + аудіо-хук + субтитр ≤8 слів
  • Carousel: sequential storytelling hook → benefits → CTA на останній картці
🎯
Retargeting — Ремаркетинг
Pixel · DPA · Customer List · Lookalike
/retargeting · platform: meta
  • Pixel audiences: ViewContent / AddToCart / InitiateCheckout / Purchase (7–180d)
  • Video engagement: 25/50/75/95% completion для різних вікон
  • Customer List з CRM (мін. 1 000 збігів) → сегменти + LAL seed
  • Ecom DPA: broad remarketing / exact product / cross-sell
  • Стратегія: MOFU (carousel + testimonial) → BOFU (urgency ≤7/7d freq cap)
  • BOFU Post-purchase upsell / cross-sell кампанія
⚗️
A/B Test — Тестування
Meta Experiments · Holdout · ABO vs CBO
/ab_test · platform: meta
  • Meta Experiments: Creative / Audience / Placement / Budget Strategy
  • Holdout Test: 10–15% audience без реклами — справжній incremental lift
  • ABO vs CBO: 14-денний тест з однаковим загальним бюджетом
  • Тести: creative format, audience (interests vs Advantage+ vs LAL vs broad)
  • Розрахунок sample size (≥50 events/7d для кожної гілки)
  • Creative framework: заміна при CTR <0.5% після ≥500 impressions
🔎
Research — Дослідження
Audience architecture · LAL seeds · Ad Library
/research · platform: meta
  • Targeting spectrum: Detailed → Broad → Advantage+ → Custom → LAL
  • Interest discovery: direct / adjacent / behavioural / competitor / publication
  • Аудиторний розмір: <100K (занадто вузько) → >50M (тільки Advantage+)
  • LAL seed ієрархія: покупці → VIP (top 20% LTV) → video 75%+ → всі відвідувачі
  • LAL розміри: 1% (якість) → 3–5% (масштаб) → 5–10% (максимальний reach)
  • Конкурентний аналіз через Meta Ad Library (формати, copy angle, CTA)
  • Output: тестова матриця 4–6 Ad Sets + 30-денний roadmap
