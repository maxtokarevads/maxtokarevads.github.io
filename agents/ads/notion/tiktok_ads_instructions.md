# TikTok Ads — Інструкції для агента

> Знання-база для навчання агента. Охоплює всі режими: plan, analyze, copy, retargeting, ab_test, budget.

---

## Платформа: загальний контекст

TikTok Ads Manager — рекламна платформа з алгоритмічним таргетингом на основі поведінки. На відміну від Meta і Google, тут аудиторія знаходиться **через контент**, а не через демографію чи інтент. Рекламу показують тим, хто взаємодіє з подібним контентом — навіть без явного налаштування таргетингу.

**Ключові відмінності від Meta/Google:**
- Алгоритм первинний — таргетинг вторинний
- Нативний UGC-стиль конвертує краще, ніж "полірована" реклама
- Звук обов'язковий (≠ Facebook де 60% дивляться без звуку)
- Перші 1–3 секунди вирішують CTR — "hook" критичніший, ніж будь-де
- Shelf life креативу: 1–2 тижні (швидше вигорає, ніж Meta)
- Events API (CAPI) ще менш зрілий ніж у Meta — враховувати over-attribution 35–55%

## Структура акаунту

```
Ad Account
└── Campaign (ціль + бюджет CBO або без)
    └── Ad Group (таргетинг + плейсмент + ставка + бюджет)
        └── Ad (креатив + текст + CTA)
```

**Campaign рівень:**
- Ціль (Objective) визначає алгоритм оптимізації
- Budget: CBO (Campaign Budget Optimization) або бюджет на Ad Group
- Smart+ Campaign: повний авто-режим (аналог ASC у Meta або PMax у Google)

**Ad Group рівень:**
- Таргетинг аудиторії
- Плейсмент (TikTok feed / Pangle / Global App Bundle)
- Стратегія ставок
- Розклад показів
- Мінімальний бюджет на навчання: **$20–50/день** на Ad Group

**Ad рівень:**
- Відео або Image (обмежено)
- Caption ≤100 символів
- CTA кнопка
- Spark Ads: окреме налаштування з органічним постом

## Режим: PLAN — Стратегія запуску

### Цілі кампаній (Objectives)

| Funnel | Objective | Оптимізація |
|--------|-----------|-------------|
| TOFU | Reach | Унікальне охоплення |
| TOFU | Video Views | ThruPlay (≥6 сек) або 2-sec views |
| MOFU | Traffic | Кліки / Landing Page Views |
| MOFU | Community Interaction | Підписки, коментарі, шери |
| MOFU | App Installs | Встановлення |
| BOFU | Website Conversions | Pixel-події (Purchase, Lead, AddToCart) |
| BOFU | Lead Generation | Вбудована TikTok Lead Form |
| BOFU | Catalog Sales | Динамічні Shopping Ads |
| BOFU | App Events | In-app конверсії |

### Типи кампаній

**Standard Campaign** — ручне керування:
- Сам встановлюєш таргетинг, ставки, плейсменти
- Підходить для тестування гіпотез і A/B тестів

**Smart+ Campaign** — автоматизований (аналог ASC Meta):
- Алгоритм сам знаходить аудиторію, плейсмент, ставку
- Рекомендується після накопичення ≥50 конверсій
- Менший контроль, але зазвичай кращі результати для e-com

**Shopping Ads** — для e-commerce:
- Підключення товарного каталогу (фід)
- Плейсменти: TikTok feed + Search + Shopping Centre
- Формати: Video Shopping Ads, Catalog Listing Ads, LIVE Shopping

### Стратегії ставок

| Стратегія | Коли використовувати | Умова |
|-----------|---------------------|-------|
| Maximum Delivery | Старт, розмасштабування | Бюджет ≥ 10× CPA за 7 днів |
| Cost Cap | Контроль CPA | Бюджет ≥ 10× bid |
| Bid Cap | Жорсткий ліміт CPC/CPM | Ризик недовитрати |
| Minimum ROAS | E-commerce з даними | Потрібна історія конверсій |

**Фаза навчання:** 7 днів або 50 конверсій — не змінювати ставки, бюджет, таргетинг.

### Таргетинг

**Demographics:**
- Вік, стать, мова, країна/місто
- Пристрій, оператор, тип з'єднання

**Interests & Behaviors:**
- Категорії інтересів (15+ вертикалей)
- Взаємодія з відео певних категорій
- Взаємодія з профілями/контентом

**Custom Audiences:**
- Website Pixel (відвідувачі сайту, події)
- Customer File (email/phone match)
- App Activity (in-app події)
- Engagement (взаємодіяли з TikTok-акаунтом/відео)
- Lead Gen Form (заповнили форму)

**Lookalike Audiences:**
- 1–3% — точний (холодний трафік)
- 5–10% — масштаб
- Джерело: Custom Audience з ≥1000 users

**TikTok-специфічне:**
- Hashtag Audience — підписники/учасники челенджу
- Creator Lookalike — аудиторія схожа на фоловерів конкретного TikToker'а

**Broad targeting (рекомендується для Smart+):**
- Мінімальний таргетинг = більше даних для алгоритму
- Особливо ефективно при великих бюджетах (>$5k/міс)

## Режим: ANALYZE — Діагностика кампанії

### Ключові метрики і норми (EU/US 2026)

| Метрика | Норма | Проблема якщо |
|---------|-------|---------------|
| CPM | $3–8 (prospecting) / $5–12 (retargeting) | >$12 — вузький таргетинг або висока конкуренція |
| CTR (In-Feed) | 1–3% | <0.5% — слабкий hook або нерелевантна аудиторія |
| Hook Rate (3-sec views / impressions) | ≥30% | <20% — перші 3 сек не чіпляють |
| Video Completion Rate | ≥25% (коротке відео) | <15% — відео нудне або надто довге |
| Avg Watch Time | ≥4–6 сек | Менше — відео не тримає увагу |
| CVR (кліки → конверсії) | 0.4–1.4% | <0.3% — проблема з лендингом або оффером |
| CPA | Залежить від ніші | Порівнювати з Meta/Google CPA |
| ROAS | >2.0 для e-com | <1.5 — збиткова кампанія |
| Engagement Rate | 3–8% | <2% — контент не резонує |

### Алгоритм діагностики

```
1. Перевір CPM → якщо аномально високий: звуж/розшир таргетинг
2. Перевір Hook Rate → якщо <20%: потрібен новий hook у перші 3 сек
3. Перевір Video Completion Rate → якщо низький: відео занадто довге або нудне
4. Перевір CTR → якщо низький при нормальному Hook: проблема з CTA або офером
5. Перевір CVR → якщо низький при нормальному CTR: проблема з лендингом
6. Перевір Frequency (якщо є) → якщо >2.5 за 7 днів: аудиторія вигорає
7. Перевір ROAS/CPA → якщо погані при хороших верхніх метриках: attribution issue
```

### Специфічні проблеми TikTok

**Creative Fatigue (вигоряння):**
- Симптом: CTR падає на 20%+ за тиждень при стабільному CPM
- Рішення: оновити hook, змінити перші 3 сек, нові відео

**Over-attribution:**
- TikTok "привласнює" конверсії, які відбулися б без реклами
- Норма over-attribution: 35–55%
- Рішення: incrementality test, порівняння з GA4 / сервером

**Learning Phase Issues:**
- Якщо Ad Group не виходить з навчання: збільш бюджет або розшир аудиторію
- Мін. 50 конверсій / 7 днів для виходу з навчання

**Pixel проблеми:**
- Перевірити Events Manager: чи коректно відстежуються події
- Якщо є розбіжність Pixel vs. реальні конверсії — підключити Events API

## Режим: COPY — Копірайтинг і скрипти

### Формати і ліміти

| Елемент | Ліміт | Примітки |
|---------|-------|---------|
| Caption | ≤100 символів | Показується під відео, часто обрізається |
| Hashtags | 3–5 тегів | Категоризація + трохи reach |
| CTA кнопка | текст з набору | Shop Now / Learn More / Sign Up / Download |
| Відео тривалість | 5–60 сек (оптимум 9–15 сек) | TopView до 60 сек |
| Аспект | 9:16 обов'язково | 1080×1920px |

### Структура відео-скрипту (Hook → Body → CTA)

**Hook (0–3 сек) — найважливіше:**
- Питання що б'є в біль: "Чому твій магазин не продає?"
- Шокова цифра: "73% людей роблять цю помилку..."
- Bold statement: "Я витратив $50k на рекламу і ось що дізнався"
- Pattern interrupt: незвичайний кут камери, рух, звук
- Текст на екрані першу секунду — обов'язково

**Body (3–12 сек):**
- Одна думка / одна перевага
- Показуй, не розповідай
- Нативний стиль (selfie camera, рух по кімнаті, реальний продукт)
- Субтитри ОБОВ'ЯЗКОВІ

**CTA (останні 2–3 сек):**
- Конкретна дія: "Тисни посилання в біо", "Свайп вгору", "Замов зараз"
- Повтори оффер: "Безкоштовно перші 30 днів"

### Шаблони скриптів по цілі

**Для Lead Gen:**
```
Hook: [Болева точка ЦА] — 1 реч.
Агітація: Ти теж [проблема]? — 1 реч.
Рішення: [Продукт] дає [результат] — 1 реч.
Соціальний доказ: [цифра] клієнтів вже — 1 реч.
CTA: Заповни форму нижче, отримай [лід-магніт]
```

**Для E-commerce:**
```
Hook: Покажи продукт у дії з сюрпризом
Body: 3 переваги → 1 секунда кожна
Proof: UGC-відгук або rating
CTA: "Кількість обмежена — посилання в біо"
```

**Для App Install:**
```
Hook: Демо найкрутішої фічі
Body: Покрокове використання (screen recording)
Social proof: "1M+ користувачів"
CTA: "Безкоштовно в App Store"
```

### Caption формула
```
[Емоційний hook 1 речення]. [Ключова вигода]. [CTA] #[нішевий тег] #[продуктовий тег] #fyp
```

Приклад: `Не витрачай час на таблиці. Наш AI робить аналіз за 30 секунд. Спробуй безкоштовно → посилання в біо 🔗 #saas #productivityhack #ai`

## Режим: RETARGETING — Ремаркетинг

### Воронка ремаркетингу TikTok

```
TOFU (холодна) → MOFU (тепла) → BOFU (гаряча)
     ↓                 ↓               ↓
 Video Views      Website Visitors   Cart Abandoners
 Profile Views    Product Viewers    Past Purchasers
 Hashtag Users    Lead Form Openers  High LTV Segment
```

### Сегменти і повідомлення

**MOFU — відвідувачі сайту (1–30 днів):**
- Custom Audience: Pixel → All Website Visitors
- Повідомлення: соціальний доказ, відгуки, порівняння
- Формат: Spark Ads (органічний відгук) або UGC
- Виключи: покупців за 30 днів

**BOFU — кинули кошик (1–7 днів):**
- Custom Audience: Pixel → AddToCart but not Purchase
- Повідомлення: нагадування + додатковий стимул (знижка, безкоштовна доставка)
- Формат: Catalog Shopping Ads (динамічний продуктовий показ)
- Ставка: Cost Cap (CPA знайомий)

**BOFU — повторні покупки (LTV):**
- Custom Audience: Purchase (90–180 днів)
- Повідомлення: нові продукти, upsell, loyalty
- Виключи: нещодавніх покупців (7–14 днів)

**Engagement Retargeting:**
- Переглянули відео ≥75% → MOFU
- Взаємодіяли з профілем → теплі потенційні покупці
- Відкрили Lead Form але не заповнили → BOFU

### Обмеження TikTok для ремаркетингу

- Pixel Custom Audiences: мінімум 1000 users для таргетингу
- Lookalike від Custom Audience: мінімум 1000 seed users (рекомендується 10000+)
- Вікно атрибуції при ремаркетингу: 7-day click / 1-day view

## Режим: AB_TEST — A/B тести

### Що тестувати в першу чергу

**Пріоритет 1: Креатив (найбільший вплив):**
- Hook варіанти (різні перші 3 сек при однаковому body)
- Стиль: UGC vs. студійне vs. motion graphic vs. screen recording
- Тривалість: 9 сек vs. 15 сек vs. 30 сек
- Субтитри vs. без субтитрів

**Пріоритет 2: Аудиторія:**
- Broad (без таргетингу) vs. Interest-based
- Custom Audience Lookalike 1% vs. 5%
- Різні вікові групи як окремі Ad Groups

**Пріоритет 3: Ставки:**
- Maximum Delivery vs. Cost Cap
- Різні значення Cost Cap

**Пріоритет 4: Плейсменти:**
- TikTok only vs. Pangle + Global App Bundle

### Правила A/B тесту TikTok

1. **Один фактор** — змінювати тільки одну змінну
2. **Мінімальний бюджет:** $50–100/день на варіант (або однаковий бюджет)
3. **Тривалість:** мінімум 7 днів (пройти фазу навчання)
4. **Статистика:** використати TikTok вбудований A/B тест (Split Test) або порівнювати вручну з поправкою на сезонність
5. **Переможця** масштабувати, тест зупинити

### TikTok Split Test (вбудований інструмент)

- Доступний для Reach / Video Views / Traffic / Conversions
- Автоматичний розподіл аудиторії без перетину
- Статистична значущість рахується платформою
- Мінімальний бюджет на тест: $100

## Режим: BUDGET — Розподіл бюджету

### Ліміти і орієнтири (EU/US SMB 2026)

| Рівень | Бюджет/міс | Що реально зробити |
|--------|-----------|-------------------|
| Мінімум для даних | $1 000–3 000 | 1–2 кампанії, базовий тест |
| SMB старт | $3 000–8 000 | Prospecting + базовий retargeting |
| SMB зріст | $8 000–20 000 | Множинні Ad Groups, активний тест |

| Mid-market | $60 000+ | TopView, Brand Takeover, Creator program |

### Розподіл бюджету по воронці

**E-commerce (рекомендований сплит):**
- 60–70% — Prospecting (холодна аудиторія, нові покупці)
- 20–30% — Retargeting (теплі + кинули кошик)
- 10% — Retention / LTV (повторні покупці)

**SaaS / Lead Gen:**
- 70–80% — Prospecting (Video Views + Lead Gen форма)
- 15–25% — Retargeting (відвідувачі сайту, незаповнені форми)
- 5% — Engagement (взаємодії з акаунтом)

### Розподіл між платформами (якщо є мульти-канал)

| Платформа | Сильний для | Типовий % бюджету |
|-----------|------------|------------------|
| TikTok | TOFU, молода ЦА (18–34), UGC, відеоконтент | 25–40% |
| Meta | Full-funnel, ремаркетинг, Lookalike | 40–50% |
| Google | Intent-based, Search, Shopping | 20–35% |

**Коли збільшити частку TikTok:**
- ЦА 18–35 років
- Продукт візуальний / демонстраційний
- Є UGC контент або можливість його генерувати
- CPM на Meta зростає (>$15)

**Коли зменшити частку TikTok:**
- ЦА 45+ (менше аудиторії)
- B2B продукт
- Немає відеоконтенту і не планується
- Attribution незрозумілий (немає CAPI)

## Специфічні формати реклами

### In-Feed Ads (основний формат)
- Тривалість: 5–60 сек (оптимум 9–15 сек)
- Показується у For You Page між органічним контентом
- Autoplay, звук за замовчуванням увімкнений
- CTA кнопка внизу праворуч
- Бюджет: від $20/день

### TopView
- Перше що бачить користувач при відкритті TikTok
- Тривалість: 5–60 сек (оптимум 15–30 сек)
- Premium форма, висока CPM
- Підходить для brand awareness, великих лончів
- Бюджет: від $50/CPM, зазвичай від $10k/кампанія

### Brand Takeover
- Повноекранний при запуску додатку, 3–5 сек
- Клікабельний (посилання або Hashtag Challenge)
- Ексклюзивно — одна категорія на день в одній країні
- Ціна: від $50k+ за день (Enterprise)

### Branded Hashtag Challenge
- UGC-механіка: бренд запускає челендж, юзери знімають відео
- Мінімум 6 днів
- Входить у Discovery page
- Підходить для TOFU awareness і залучення
- Бюджет: від $100k+ (включає підтримку TikTok команди)

### Spark Ads (буст органіки)
- Просуваєш органічний пост (свого акаунту або creators')
- Зберігає лайки, коменти, шери на оригінальному відео
- Ефективний для social proof
- Вимагає авторизацію від власника акаунту

### Shopping Ads (для e-commerce)
- **Video Shopping Ads:** відео + продуктові картки внизу
- **Catalog Listing Ads:** картки з фіду без відео
- **LIVE Shopping Ads:** просування LIVE трансляцій з продажами
- Потребує підключеного каталогу в TikTok Shop або Catalog Manager

## Технічні вимоги

### Відео
- Формат: MP4 або MOV
- Розміщення: 9:16 (вертикальне) — ОБОВ'ЯЗКОВО
- Роздільна здатність: мінімум 720×1280, рекомендовано 1080×1920
- Розмір файлу: до 500MB
- Тривалість: 5–60 секунд
- FPS: 23–60

### Зображення (обмежено)
- JPEG або PNG
- Розмір: ≥1200×1500px для вертикального
- Підтримується тільки для деяких плейсментів (Pangle)

### Pixel і відстеження
- TikTok Pixel: стандартні події (ViewContent, AddToCart, Purchase, Lead, CompleteRegistration)
- Events API (CAPI): рекомендується для iOS 14.5+ аудиторії
- UTM параметри: обов'язково для cross-platform аналітики
- Вікно атрибуції: 7-day click / 1-day view (стандарт)
